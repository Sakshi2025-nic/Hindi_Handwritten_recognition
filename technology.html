Technology Stack & Methodology

Core Components

1. Deep Learning Model: Handwritten Hindi Text Recognition

The core of the system is a deep neural network designed for accurate handwritten Hindi text recognition. The architecture typically comprises:

  • Convolutional Neural Network (CNN) Layers: Feature extraction from input images, learning key visual features, especially for handwritten data.
  • Recurrent Neural Network (RNN) Layers (LSTM/GRU): Capturing sequential dependencies in the text, enabling the model to understand context within the word. Bidirectional RNNs are used for better performance.
  • Connectionist Temporal Classification (CTC) Loss: Enables training the model for sequence recognition, without needing to segment individual characters.

Framework: TensorFlow 1.15 (as indicated in logs). Model specifics and detailed architecture can be found in [Link to your Model Architecture/Paper, if available].

2. Image Preprocessing

Preparing the images for optimal model performance. Preprocessing steps include:

  • Grayscale Conversion: Simplifying the image by converting it to a single channel.
  • Resizing: Standardizing image dimensions, typically to a fixed height while maintaining aspect ratio.
  • Normalization: Scaling pixel values for consistent input to the model.
  • Binarization: Applying a threshold to convert the image to black and white, enhancing the contrast of the text. This step is crucial for cleaning up the handwriting and removing noise.

Primary Library Used: OpenCV (`cv2`)

3. Hindi-to-English Translation

The system integrates Hindi-to-English translation to provide clear and understandable output to a broader audience. The key aspects of this component include:

  • Translation API: Integration with the Google Translate API , to translate recognized Hindi words/sentences into other languages.
  • Text Processing: Handling the text input from the OCR module before sending it to the translation API, which might involve removing any unusual characters or formatting.
  • Display of Results: Displaying the translated text alongside the recognized Hindi text.

4. Gender Prediction

The project incorporates a gender prediction component, adding an analytical layer to the text recognition. Implementation details:

  • Feature Extraction: We use custom feature extraction techniques tailored for Devanagari handwriting, including stroke curvature, slant direction, pressure patterns, and character formation structures to assist in gender prediction.
  • Model: We developed and trained our own models for both OCR and gender prediction. The OCR system uses a CNN-LSTM architecture, while the gender prediction model is a separate deep learning classifier integrated into the system pipeline.
  • Output: The system predicts the gender (male/female/other) of the writer with a confidence score, alongside the recognized Devanagari word from the handwritten image.

5. Backend Framework

The server-side logic, API handling, and coordination are managed by **Flask**, a Python microframework selected for its simplicity and efficiency. Flask's roles include:

  • Defining URL routes (e.g., /, /recognize, /predict, /api/examples).
  • Handling HTTP requests (GET for pages, POST for predictions).
  • Receiving and securely saving uploaded image files.
  • Orchestrating the workflow: calling functions to handle image preprocessing, performing handwriting recognition, translating results, predicting gender, and returning the aggregated results.
  • Formatting the results into JSON for the API endpoint.
  • Rendering HTML templates (using Jinja2) to display web pages.
  • Serving static assets: CSS, JavaScript, and images.

6. Frontend Interface

The application offers a user-friendly web interface built using:

  • HTML5: The structural basis of the web pages.
  • Tailwind CSS: A utility-first CSS framework to ensure a clean and consistent design.
  • Vanilla JavaScript: Providing client-side interactivity with functionality, including:
    • Managing file uploads and the display of image previews.
    • Initiating API calls to the Flask backend for recognition.
    • Receiving and handling the JSON response.
    • Dynamically updating the UI to display prediction results, loading indicators, or error messages.
    • Handling the basic UI state changes.
  • Remixicon / Google Fonts: Incorporating the UI elements for icons and fonts.
